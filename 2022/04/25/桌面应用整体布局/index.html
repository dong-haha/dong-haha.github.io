桌面应用整体布局

发表于 2022-04-25 分类于 前端 ， 桌面应用开发

1. 技术选取

  1. vue2+element ui
  2. electron-egg =2.0
  3. express = 4.17
  4. R = 4.1

2. 详细介绍

  1. 首先用vue搭建前端页面，其实也是在原有ee框架的模板上改动。

    把原项目的ant ui改成 element ui，我习惯用后者。然后就是路由和子路由

  2. vue与electron通信，用的 IpcRenderer，原框架作者封装的，原理咋也不懂，用就完了。框架提供了很多通信demo,后期开发很方便

  3. express来写vue用的接口，因为本来就是node环境，直接用node写接口多方便，不用java和PHP

  4. 整个项目用的electron比较少，主要是用它提供的桌面应用的壳子，和平时web开发差不多，只有一些文件操作用electron调用node的fs模块

其他

  1. 在vue中全局引入global变量，来告诉每个组件当前ip。

    软件默认是本地运行的，express开在本地的8333端口，用户可以改成远程ip，然后把express代码原封不动（真的啥也没改），放到服务器运行，就行了。就相当于把后端搬到用户桌面运行了。

  2. express框架。首先，ee框架是自带http服务的，可以直接用它的写后端。但是因为我还要在自己服

务器运行，就不能用它的框架写。这样就只用写一份，两地运行了。
但是测试时，我都是先开npm run serve 开vue，再node ./bin/www开express，这合到一起，问题就是，这后端代码放哪，在哪引入啊。
我也对node不熟，我乱试也成功了。在整个ee框架的main.js的new Main(),加上require就行了

当然，这确实把项目跑起来了，但是不好。
整个项目是以new Main为入口的，我相当于又开了一个程序。ee框架作者封装的所有功能都用不了。 好难受啊，特别是日志文件和app全局变量，调试时很难。我这express相当于是孤立的，只能自己用原生node的api （当然这的唯一好处就是之前说的：服务器上直接运行，因为没有依赖其他框架）

  1. R环境（最重要）

我发现，一个应用安装，实际上先把相关代码解压到安装路径，然后创建一些桌面快捷图标，向windows注册，写环境变量等其他。这一点很重要，是我项目能跑下来的核心，我发现，把R安装到电脑上后，找到R安装位置，把它整体移动到其他地方，甚至装到U盘插到一台没有R环境的电脑，用R.exe和Rscript.exe也能运行。
正因如此，我才决定开发这个软件。R环境是整个项目最大的技术障碍，如果不能保证其他计算机环境能运行R，我这软件就没啥用。我可以在安装包中内置个R，用Rscript.exe ***.R 来运行R代码，这个问题在3.29号被发现，在之前，我都没想过要参加比赛(计算机程序设计大赛的作品)。这个问题解决后，其他的都是小技术障碍，慢慢解决就行。
